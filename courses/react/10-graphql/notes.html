GraphQL.

Названия запросов другие, поля другие - это на каждом бэки свои query - получение данных (чтение с бэка) mutation - изменение данных, бэк будет что-то менять (изменение бэка) 
query cats($q:String) {
  CategoryFind(query:$q){
  	name, goods {  name}
  }
}
CategoryFind - имя операции. Некая ф-ция на бэке () - какие-то конкретные значения, нужные для ф-ции {} - то что нам прислать в ответ (проекция того, что бы мы хотели получить) ========= fetch http://shop-roles.asmer.fs.a-level.com.ua/graphql method: POST content-type: application/json body: json c query - строка и variables - объект с вложенными параметрами
