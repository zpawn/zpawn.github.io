GraphQL

GraphQL это синтаксис, который описывает как запрашивать данные

Основные характеристики

  • Позволяет клиенту точно указать, какие данные ему нужны.
  • Облегчает агрегацию данных из нескольких источников.
  • Использует систему типов для описания данных.

Анатомия запроса

Query - получение данных

            query log{
              login(login:"test", password:"test")
            }
  • log - имя для вашего удобства
  • login - имя запроса на бэке.
    • login - строка логина
    • password - пароль

Mutation - изменение данных

            mutation reg{
              UserUpsert(user: {login:"test", password:"test"}){
                _id, login
              }
            }
  • reg - имя для вашего удобства
  • UserUpsert - имя мутации с бэка
  • user - значение типа для отправки на сервер (input UserInput).
  • _id, login - поля которые получим в ответ.
