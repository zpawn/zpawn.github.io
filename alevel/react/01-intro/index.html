React

JavaScript-библиотека для создания пользовательских интерфейсов.

Реакт позволяет построить абсракцию вокруг верстки. Т.е. вы вместо того чтоб работать примитивами уровня HTML, вы оперируете отдельными частями приложения и говорите что вы хотите видеть, а не пошагово описываете как должно выглядеть.

Компоненты

Компоненты позволяют разбить интерфейс на независимые части, про которые легко думать в отдельности. Их можно складывать вместе и использовать несколько раз.

Во многом компоненты ведут себя как обычные функции JavaScript. Они принимают произвольные входные данные (так называемые «пропсы») и возвращают React-элементы, описывающие, что мы хотим увидеть на экране.

Создаем приложение.

Create React App is an officially supported way to create single-page React applications. It offers a modern build setup with no configuration. 

            npx create-react-app my-app
            cd my-app
            npm start

Структура проекта:

  • src - в этой папке будут все наши компоненты
  • public - в этой папке хранится наш шаблон html странички.

Как отрендерить компонент

            ReactDOM.render(
              <h1>Hello, World!</h1>, // react-элемент
              document.getElementById('root')
            );
             
            ReactDOM.render(
              createElement('h1', {}, 'Hello, World!'), // react-элемент
              document.getElementById('root')
            );

const element = <h1>Hello, World</h1> - react-элемент. В отличие от DOM-элементов, элементы React — это простые объекты, не отнимающие много ресурсов. React DOM обновляет DOM, чтобы он соответствовал переданным React-элементам. 

            {
              type: 'h1',
              props: {
                children: 'Hello, World!'
              }
            };

Функциональные компоненты

            function Welcome(props) {
              return <h1>Привет, {props.name}</h1>
            }

Эта функция — компонент, потому что она получает данные в одном объекте («пропсы») в качестве параметра и возвращает React-элемент. Мы будем называть такие компоненты «функциональными», так как они буквально являются функциями.

Классовые компоненты

Ещё компоненты можно определять как классы ES6: 

            class Welcome extends React.Component {
              render() {
                return <h1>Привет, {this.props.name}</h1>
              }
            }

Как создать собственный компонент

Пока что мы только встречали React-элементы, представляющие собой DOM-теги:

const element = <div />

Но элементы могут описывать и наши собственные компоненты:

const element = <Welcome name="Тарас" />

Пропсы

Когда React встречает подобный элемент, он собирает все JSX-атрибуты и дочерние элементы в один объект (props) и передаёт их нашему компоненту. 

            function Welcome(props) {
              return <h1>Привет, {props.name}</h1>
            }
            ReactDOM.render(<Welcome name="Тарас" />, ...)

Ньюансы

  • Т.к. JSX компилируется в вызовы функции React.createElement, библиотека React всегда должна находиться в области видимости вашего кода JSX. import React from 'react';
  • Большая буква - значит компоненьт и React будет в текущей области видимости искать эту ф-цию
    Маленькая буква - dom-элемент
  • Использование записи через точку "." 
                        const MyComponents = {
                      DatePicker: (props) => <div> ... </div>
                    }
                    const BlueDatePicker = () => {
                      return <MyComponents.DatePicker color="blue" />
                    }

Свойства props в JSX

  • JavaScript выражения как свойства.
    <MyComponent foo={1 + 2 + 3 + 4} />
  • Строковые литералы
    <MyComponent message="Привет мир!" />
    <MyComponent message={'Привет мир!'} />
  • Инструкции if или циклы for не являются выражениями в JavaScript, поэтому они не могут использоваться непосредственно в JSX. 
                        function NumberDescriber(props) {
                      let description;
                      if (props.number % 2 == 0) {
                        description = <strong>even</strong>
                      } else {
                        description = <i>odd</i>
                      }
                      return <div>{props.number} - это {description} число</div>
                    }
  • Установка свойств по умолчанию в true
    <button disabled />
    <button disabled={true} />
  • Spread – атрибуты 
                        const Button = props => {
                      const { type, ...other } = props;
                      const className = type === "primary" ? "PrimaryBtn" : "SecondaryBtn";
                      return <button className={className} {...other} />
                    };
  • Booleans, Null и Undefined игнорируются
                        <div />
                    <div></div>
                    <div>{null}</div>
                    <div>{undefined}</div>
                    <div>{false}</div>
                    <div>{true}</div>

Children (потомки)

  • Несколько Компонентов
                        <MyContainer />
                      <MyFirstComponent />
                      <MySecondComponent />
                    </MyContainer>
  • JavaScript выражения как потомки
    <MyComponent>{'Привет!'}</MyComponent>
  • Функции как потомки 
                        function UserList(props) {
                      return <ul>{props.users.map((user) => props.children(user))}</ul>
                    }
                    function UserPage() {
                      const users = [{id: 1, name: 'Вася'}, {id: 2, name: 'Петя'}];
                      return (
                        <UserList users={users}>
                          {(user) => <li key={user.id}>Пользователь: {user.name}</li>}
                        </UserList>
                      );
                    }
